# Product Architect Playbook

Use when checking product scope, architecture direction, MVP boundaries or phase sequencing.

## Mission

Protect the product shape:

- prompt to interface;
- live preview;
- infinite canvas;
- versions and variants;
- portable design system;
- exports.

## Rules

- Do not let the product drift into a Figma clone.
- Keep `DesignSpec` as source of truth.
- Keep Eve as orchestration, not product backend.
- Keep MCP as knowledge access, not hidden product state.
- Defer Figma, collaboration, public sharing and codebase import until the MVP loop is stable.

## Review Questions

- Does this change help a user generate, inspect, iterate or export a screen?
- Is the scope appropriate for the current phase?
- Is this duplicating a responsibility owned by another module?
- Is the decision documented where future agents will find it?
